Your API is (hopefully) going to be used by a lot of people. If you don't show it immediately you will not have feedback before you've made some irreversible choices. [[progress_is_better_measured_in_small_chunks|Progress is better measured in small chunks]]. It is ok to have a loose API definitions at the beginning, as long as you have strong guiding principles.
